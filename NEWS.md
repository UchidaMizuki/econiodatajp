# econiodatajp 0.1.0

* Fixed several `region_class = "pref"` (2015) source downloads: Tochigi's URL pointed at Ibaraki's own workbook (wrong prefecture entirely), Kumamoto's URL pointed at the 2020 table instead of 2015, and Miyagi's, Kanagawa's, Shiga's, Yamaguchi's, Nagasaki's, and Nara's URLs had gone stale and 404'd.
* Tarchive target names now lead with `region` (e.g. `iotable_01_hokkaido_105_producer_price_competitive_import_ja`, or `iotable_nation_medium_producer_price_competitive_import_ja` for a table whose scope is the whole country) instead of trailing it or omitting it, matching `io_table_get()`'s own argument order (`region` before `sector_class`); `io_table_available()`'s columns are reordered to match (`region_type`, `region_class`, `year`, `region`, `sector_class`, `price_type`, `import_type`, `language`, `name`). This only affects internal tarchive target names and `io_table_available()`'s column order, not `io_table_get()`/`io_table_target()`'s own arguments.
* `io_table_available()` now also lists `region_class = "pref"`, `region_type = "regional"` tables (one row per prefecture), previously omitted because their target names didn't match the naming scheme it introspects.
* `io_table_available()` is now `io_table_available`, a tibble shipped as package data rather than a function -- introspecting every tarchive pipeline was too slow to redo on every call, including from inside a `_targets.R` pipeline (e.g. `tarchetypes::tar_eval(..., values = io_table_available)`), which re-sources on every run. This is a breaking change: drop the `()`. The shipped data is checked against a live recomputation by a test, so it can't silently go stale.
* `io_table_get()`/`io_table_target()`/`io_table_target_raw()`'s `year` argument now comes before `region`, matching the unified `region_type`-`region_class`-`year` tarchive pipeline naming scheme every pipeline now follows (previously `region_type = "multiregional"` pipelines had an extra `-nation-` segment, e.g. `iotable-multiregional-pref-nation-2011`, now `iotable-multiregional-pref-2011`). This is a breaking change for any positional (unnamed) call.
* `io_table_get()`/`io_table_target()`/`io_table_target_raw()`'s `competitive_import` (a logical) has been replaced by `import_type` (`"competitive_import"` or `"noncompetitive_import"`), matching the string-enum style of `price_type`/`sector_class`/`language` and `io_table_available()`'s other columns (its own `competitive_import` column is renamed to `import_type` and is now a character column, not logical). Every tarchive target also always carries an explicit `_competitive_import`/`_noncompetitive_import` marker in its name (e.g. `iotable_nation_medium_producer_price_competitive_import_ja`), matching the always-explicit `_ja`/`_en` language suffix, instead of leaving competitive-import as an unmarked, implicit default. This is a breaking change: replace `competitive_import = TRUE`/`FALSE` with `import_type = "competitive_import"`/`"noncompetitive_import"`.
* `io_table_get()`/`io_table_target()`'s `region` no longer accepts a numeric prefecture code (e.g. `1`) as a shorthand for the code_name fragment (e.g. `"01_hokkaido"`) -- only the exact code_name fragment used in the archive is accepted now. The numeric-code-to-name lookup this shorthand needed is deferred to a future `jpcity`-based resolver rather than kept as a stopgap in this package.
* `io_table_get()`/`io_table_target()`'s `region` now defaults to `"nation"` internally rather than being left absent -- every table covers *some* region, and "the whole nation as a single scope" is itself a region, not an absence of one. `region_class = "pref"`/`"block"` under `region_type = "multiregional"` also resolve to `"nation"` now, since their prefecture/block breakdown is `region_class` detail, not a different `region` scope. `io_table_available`'s `region` column is never `NA` as a result -- it's `"nation"` for every table except a `region_class = "pref"`, `region_type = "regional"` one, where it's a real per-prefecture code_name fragment. `region` still isn't required for these tables (`NULL`, the default, is still accepted).
* `io_table_get()`/`io_table_target()`'s `sector_class` for `region_class = "pref"` and `"block"` tables is now each table's actual official sector count (e.g. `"105"` for Hokkaido's own table, `"26"`/`"31"` for the 2005/2011 interregional table) instead of a `"small"`/`"medium"`/`"large"` label borrowed from the national table's classification tiers, which don't exist for these tables and pointed in a misleading direction (e.g. Hokkaido's old `"medium"` label had fewer sectors than some other prefectures' old `"small"`).
* `io_table_get()`/`io_table_target()` now suggest the closest valid pipeline or archive name (via `tarchives::tar_check_archive_pipeline()`/`tar_check_archive_name()`) when an invalid `region_type`/`region_class`/`year`/`sector_class` combination is requested, instead of only listing every valid choice.
